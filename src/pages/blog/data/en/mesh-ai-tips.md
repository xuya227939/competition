## Prompt Writing Techniques

### 1. Structured Description Method

**Basic Structure:**

**Example Comparison:**

- ❌ Poor description: "a chair"
- ✅ Good description: "a modern-style black leather office chair with metal frame and adjustable armrests"

### 2. Keyword Usage Strategy

**Shape Keywords:**

- Basic shapes: cube, cylinder, sphere, cone
- Composite shapes: L-shaped, U-shaped, ring-shaped, spiral
- Organic shapes: streamlined, curved, irregular

**Material Keywords:**

- Metals: stainless steel, copper, iron, aluminum, gold, silver
- Woods: oak, pine, walnut, cherry, bamboo
- Stones: marble, granite, limestone, sandstone
- Fabrics: cotton, silk, leather, felt, canvas

**Style Keywords:**

- Era styles: modern, classical, medieval, futuristic
- Art styles: minimalist, baroque, industrial, natural
- Cultural styles: Nordic, Japanese, American, European

### 3. Detail Description Techniques

**Size Description:**

- Use specific values: "height 30cm, width 50cm"
- Use relative descriptions: "small, medium, large"
- Use ratio descriptions: "aspect ratio 2:1"

**Texture Description:**

- Surface texture: "smooth, rough, matte, glossy"
- Pattern texture: "striped, checkered, spotted, geometric pattern"
- Color gradient: "dark to light, gradient effect"

**Function Description:**

- Purpose: "office use, home use, decorative use"
- Special features: "adjustable, foldable, rotatable"

## Parameter Adjustment Optimization

### 1. Quality Parameter Settings

**Generation Quality Levels:**

- **Standard Quality**: Suitable for rapid preview and concept validation
- **High Quality**: Suitable for most application scenarios
- **Ultra High Quality**: Suitable for professional use and final output

**Recommended Settings:**

- Concept phase: Use standard quality for rapid iteration
- Development phase: Use high quality for detailed design
- Final phase: Use ultra high quality for finished products

### 2. Style Parameter Adjustment

**Style Intensity:**

- **Low Intensity**: Maintain original features, slight stylization
- **Medium Intensity**: Balance style and practicality
- **High Intensity**: Strong stylization, prominent artistic sense

**Style Consistency:**

- Ensure overall style unity
- Avoid style conflicts
- Maintain brand tone

### 3. Complexity Control

**Geometric Complexity:**

- **Simple**: Basic geometry, suitable for rapid prototyping
- **Medium**: Moderate details, suitable for most applications
- **Complex**: Rich details, suitable for high-quality output

**Material Complexity:**

- Choose appropriate material complexity based on purpose
- Balance visual effects and performance needs
- Consider rendering and export efficiency

## Input Optimization Strategy

### 1. Reference Image Usage

**Image Selection Principles:**

- Choose high-quality, clear reference images
- Ensure images are relevant to target models
- Avoid overly complex background interference

**Image Preprocessing:**

- Adjust image size and proportions
- Enhance contrast and clarity
- Remove unnecessary background elements

### 2. Multi-angle References

**Viewpoint Selection:**

- Front view: Show main features
- Side view: Show silhouette and proportions
- Top view: Show overall layout

**Reference Combination:**

- Combine multiple angle reference images
- Provide complete visual information
- Improve generation accuracy

### 3. Text Description Optimization

**Description Length Control:**

- Avoid overly long descriptions (recommend 50-200 words)
- Highlight key features
- Use concise and clear language

**Description Logic:**

- Sort descriptions by importance
- Maintain description logic
- Avoid contradictory information

## Common Problem Solutions

### 1. Generation Quality Issues

**Problem Diagnosis:**

- Check if prompt descriptions are clear
- Confirm if parameter settings are appropriate
- Verify input image quality

**Solutions:**

- Optimize prompt descriptions
- Adjust generation parameters
- Replace with high-quality reference images

### 2. Style Inconsistency

**Problem Causes:**

- Prompt descriptions not specific enough
- Style parameter settings inappropriate
- Reference image style conflicts

**Solutions:**

- Clarify style requirements
- Unify style parameters
- Choose consistent reference images

### 3. Missing Details

**Problem Analysis:**

- Complexity settings too low
- Descriptions not detailed enough
- Insufficient reference image information

**Improvement Measures:**

- Increase complexity settings
- Add detail descriptions
- Provide more reference information
